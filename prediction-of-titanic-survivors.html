Dealing with Missing Values

Before actually taking a look at the missing values, let's create a utility function to print the missing values, as we're surely going to need it later on. 

def null_table(training, testing):
  print("Training dataset missing values:\n{}".format(pd.isnull(training).sum()))
  print("\nTesting dataset missing values:\n{}".format(pd.isnull(testing).sum()))

Seems like the training dataset contains 177 missing values for the 'Age' attribute, 687 missing values for 'Cabin' and 2 for 'Embarked', while the testing dataset has 86 missing values for 'Age' and 327 for 'Cabin'. Put in another way, we have ~77% of missing values for 'Cabin' in training and ~78% for the same attribute in the testing dataset. Meanwhile, the 177 missing values of 'Age' in training constitute a ~20% of the dataset and the 86 missing ones in testing also constitute a ~20% of the testing dataset.

Getting rid of the examples that have a missing value on either 'Cabin' or 'Age' is not an option, as we would be wasting a ton of information. On the other hand, getting rid of the 2 examples that have a missing value on 'Embarked' in the training dataset, is actually something that we can do, since we're only getting rid of 0,2% of the training dataset, the same reasoning applies to 'Fare' on the testing dataset.

We'll get rid of 'Cabin' due to its missing values and 'Ticket' because it's too noisy.

Another option here to handle the missing values is to perform imputation on the attributes, that's what we're going to do for 'Age' and 'Embarked', we'll first take a look at the mode of 'Embarked' as we'll replace the missing values with it. 

# Inspect count of each value of 'Embarked'

644 out of the 891 examples of the training dataset have the value 'S' on 'Embarked', which means this is the mode of this attribute. Now that we know this, let's drop the 'Cabin' and 'Ticket' attributes and perform imputation on 'Embarked' by replacing its missing values with the mode. 

# Drop Cabin and Ticket attributes
training.drop(labels = ["Cabin", "Ticket"], axis = 1, inplace = True)
testing.drop(labels = ["Cabin", "Ticket"], axis = 1, inplace = True)

# Fill 'Embarked' with its mode
training["Embarked"].fillna("S", inplace = True)

Model Fitting, Optimizing, and Predicting

We finished with data preprocessing, so now we try out different models and compare their performance. Let's define the training features set, as well as the training label set. 

X_train = training.drop(labels=["PassengerId", "Survived"], axis=1) # define training features set
y_train = training["Survived"] # define training label set

We'll create a validation dataset to test the general accuracy of our model. We use a 0.8 / 0.2 ratio. 

X_training, X_valid, y_training, y_valid = train_test_split(X_train, y_train,
                                                            test_size=0.2,
                                                            random_state=0)

Now that our validation dataset is ready, we proceed to train each model. For now, I'll provide the code blocks needed for training, we'll see the accuracy results for each model in the following section.

# SVC Model
svc_clf = SVC()

parameters_svc = {"kernel": ["rbf", "linear"], "probability": [True, False], "verbose": [True, False]}

grid_svc = GridSearchCV(svc_clf, parameters_svc, scoring=make_scorer(accuracy_score))
grid_svc.fit(X_training, y_training)

svc_clf = grid_svc.best_estimator_

svc_clf.fit(X_training, y_training)
pred_svc = svc_clf.predict(X_valid)
